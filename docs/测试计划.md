● 本地测试UX与故事质量执行计划

  基于对旧报告的深度分析，我制定了一个系统性的本地测试计划，重点关注睡前场景的用户体验和故事质量。

  Phase 1: 环境准备与基础验证

  1.1 启动本地开发环境

  # 1. 确保依赖完整
  npm run install:all

  # 2. 启动开发服务器
  npm run dev

  # 3. 验证基础功能
  # - 前端: http://localhost:3000
  # - 后端: http://localhost:5000

  1.2 验证Mock数据质量

  - 测试无API密钥情况下的故事生成
  - 确认Mock数据是否按10岁儿童标准优化
  - 检查不同主题的内容差异化

  Phase 2: 睡前场景UX测试

  2.1 阅读体验测试（核心）

  测试重点：根据旧报告，800-1200字对睡前过长

  测试方法：
  1. 生成不同主题的故事片段
  2. 实际阅读计时（模拟10岁儿童阅读速度）
  3. 评估认知负荷和注意力集中度

  评估标准：
  - 单次阅读时间是否超过2-3分钟？
  - 内容是否过于密集需要分段？
  - 字体大小和间距是否适合移动设备？

  2.2 选择设计测试

  当前问题：渐进式3选 vs 故事树2选不一致

  测试内容：
  - 比较3选择 vs 2选择的决策压力
  - 评估选择文案的差异化程度
  - 检查选择是否具有真实的策略意义

  2.3 情绪曲线测试

  睡前特殊要求：低刺激、收束感、安抚效果

  测试方法：
  - 完整体验一个故事流程
  - 关注故事的紧张-缓解节奏
  - 评估结尾的温暖收束效果

  Phase 3: 交互流程深度测试

  3.1 双模式对比测试

  测试场景：
  - 渐进式模式：完整体验8-10轮交互
  - 故事树模式：体验3轮选择到结局

  对比维度：
  - 等待时间感受（渐进式每次等待 vs 树模式一次等待）
  - 沉浸感差异
  - 睡前场景的适用性

  3.2 进度反馈测试

  当前状态：
  进度：第 X / Y 次互动

  测试要点：
  - 进度显示是否给人安全感？
  - 接近结尾时是否有适当提示？
  - 孩子能否预期故事何时结束？

  3.3 反馈机制测试

  鼓励语系统：
  const encouragements = [
    '太棒了！继续你的故事冒险吧！',
    '你的想象力真丰富！',
    // ...8种鼓励语
  ];

  测试内容：
  - 鼓励语是否会重复？
  - 是否与选择内容相关？
  - 频率是否合适？

  Phase 4: 视觉设计睡前适配测试

  4.1 色彩与动效测试

  睡前要求：低刺激、由动转静

  测试重点：
  - 当前的粉彩色调是否适合睡前？
  - 动画效果是否过于活跃？
  - 是否需要夜间模式？

  4.2 可读性测试

  移动设备场景：
  - 字体大小在手机上的实际效果
  - 行距是否适合儿童阅读习惯
  - 按钮大小是否适合儿童操作

  Phase 5: 故事内容质量深度评估

  5.1 年龄适配性测试

  使用旧报告的评分标准：

  内容评分量表（关键项）：
  1. 年龄适配：语域、词汇与句式适合8–12岁 ⭐⭐⭐⭐⭐
  2. 选择互异：策略/价值/后果差异显著 ⭐⭐⭐⭐⭐
  3. 教育元素：每段≥1个知识/文化/品格点 ⭐⭐⭐⭐⭐
  4. 情绪曲线：张弛有度，睡前收束温暖 ⭐⭐⭐⭐⭐

  5.2 主题多样性测试

  测试不同主题的生成质量：
  - 太空主题：科学教育融入效果
  - 独角兽主题：品格教育体现
  - 通用主题：教育价值是否足够

  5.3 连贯性测试

  长期互动质量：
  - 多轮选择后故事逻辑是否一致？
  - 角色性格是否保持？
  - 世界观设定是否稳定？

  Phase 6: 问题记录与改进建议

  6.1 建立测试记录表格

  | 测试项目 | 当前表现 | 问题发现 | 改进建议 | 优先级 |
  |---------|---------|---------|---------|---------|
  | 阅读时长 |  |  |  | P0 |
  | 选择设计 |  |  |  | P0 |
  | 情绪收束 |  |  |  | P0 |

  6.2 分类改进建议

  P0级（影响基础体验）：
  - 文本长度优化
  - 选择数量统一
  - 情绪安抚增强

  P1级（提升睡前体验）：
  - 家长参与功能
  - 睡前仪式感
  - 进度可预期性

  P2级（长期优化）：
  - 夜间模式
  - 护眼提示
  - 个性化推荐

  执行时间安排

  总计预估时间：4-6小时

  - Phase 1：30分钟 - 环境搭建
  - Phase 2：90分钟 - 睡前UX核心测试
  - Phase 3：60分钟 - 交互流程测试
  - Phase 4：30分钟 - 视觉适配测试
  - Phase 5：90分钟 - 内容质量深度测试
  - Phase 6：30分钟 - 总结与建议

  测试输出

  1. 详细测试报告：记录每个测试项的具体表现
  2. 问题优先级清单：按P0/P1/P2分类的改进建议
  3. 用户体验评分：使用旧报告的量化标准
  4. 下一步行动计划：基于测试结果的具体改进方案
