

     ZenMCP Spec-driven API/Backend Development Plan                                                                            
                                                                                                                                
     Phase 1: Requirements & Architecture Analysis                                                                              
                                                                                                                                
     1. Use ThinkDeep for Requirements Analysis                                                                                 
       - Analyze business requirements and technical constraints                                                                
       - Compare architectural approaches (REST vs GraphQL, monolith vs microservices)                                          
       - Use high or max thinking mode for complex decisions                                                                    
       - Example: "Think deeper about API authentication strategy for multi-tenant SaaS with gemini pro using max thinking mode"
     2. Use Planner for Development Roadmap                                                                                     
       - Create structured implementation plan                                                                                  
       - Break down complex features into manageable tasks                                                                      
       - Plan API endpoints, data models, and integration points                                                                
                                                                                                                                
     Phase 2: Design & Specification                                                                                            
                                                                                                                                
     3. Use ThinkDeep for API Design Decisions                                                                                  
       - Analyze trade-offs between different API patterns                                                                      
       - Validate data modeling decisions                                                                                       
       - Consider security and performance implications                                                                         
       - Example: "Use o3 to think deeper about database schema design for user management system"                              
     4. Use Analyze for Existing Codebase Integration                                                                           
       - Understand current architecture and patterns                                                                           
       - Identify integration points and potential conflicts                                                                    
       - Analyze dependencies and coupling                                                                                      
                                                                                                                                
     Phase 3: Implementation & Validation                                                                                       
                                                                                                                                
     5. Use TestGen for Comprehensive Testing                                                                                   
       - Generate unit tests, integration tests, and edge cases                                                                 
       - Create API endpoint tests and validation scenarios                                                                     
       - Example: "Create thorough tests for the authentication API in auth/routes.py, focus on edge cases and security"        
     6. Use CodeReview for Quality Assurance                                                                                    
       - Review implementation against specifications                                                                           
       - Validate security best practices                                                                                       
       - Ensure code quality and maintainability                                                                                
     7. Use SecAudit for Security Validation                                                                                    
       - Perform OWASP security analysis                                                                                        
       - Validate API security patterns                                                                                         
       - Check for common vulnerabilities                                                                                       
                                                                                                                                
     Key ZenMCP Features for Spec Development:                                                                                  
                                                                                                                                
     - Multi-model orchestration: Automatically selects best AI model for each task                                             
     - Large context windows: Can analyze entire codebases (especially with Gemini Pro)                                         
     - Visual analysis: Can analyze architecture diagrams and system designs                                                    
     - Web search integration: Access to current best practices and documentation                                               
     - Continuation support: Multi-turn conversations for complex decisions                                                     
                                                                                                                                
     Configuration Requirements:                                                                                                
                                                                                                                                
     - Set up API keys for desired models (Gemini, OpenAI, etc.)                                                                
     - Configure custom models if using local inference                                                                         
     - Use model aliases in conf/custom_models.json for easy reference
